# Mem0 v2.0 API Compatibility Fix (ValueError: frozenset)

## Problem
Tools `search_memory` and `get_context` crash with error:
`ValueError: Top-level entity parameters frozenset({'user_id'}) are not supported in get_all(). Use filters={'user_id': '...'} instead.`

## Root Cause
`mem0ai` v2.0.0+ changed its core API. Methods like `get_all()` and `search()` no longer accept `user_id`, `agent_id`, or `run_id` as top-level keyword arguments. These must now be encapsulated within a `filters` dictionary.

## Solution
Updated `src/mem0_local_mcp/mem_manager.py` to move `user_id` into the `filters` argument for both `search` and `get_all`.

### Before
```python
# get_all
all_mems = self.memory.get_all(user_id=self.user_id)

# search
results = self.memory.search(query, filters=search_filters) # where search_filters included user_id
```

### After
```python
# get_all
results = self.memory.get_all(filters={"user_id": self.user_id})

# search
filters = {"user_id": self.user_id}
if category:
    filters["category"] = category
results = self.memory.search(query, filters=filters, top_k=limit)
```

## Updated Findings (Post-Implementation)
- **Indentations Matter**: When cleaning up debug logs, ensure `return` statements are correctly indented within `with self.lock:` blocks to avoid `IndentationError` that crashes the MCP server.
- **Inference Stability**: Setting `infer=False` in `memory.add()` ensures that memories are stored literally without LLM-based fact extraction, which can sometimes discard context or fail silently if the extraction result is empty.
